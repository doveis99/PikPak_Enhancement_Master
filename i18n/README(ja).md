## ✨ 主な機能 (Features)

### ✨ 操作体験とナビゲーションエンジン

* **UI 再構築**：公式機能をベースに、全体の操作体系を **Windows ファイルエクスプローラー** に近い感覚で再構築し、操作フローをより直感的にしました。
* **ターボモード**：有効化すると Web 側のロジックを深く引き継ぎ、大量ファイル環境でのカクつき、クラッシュ、メモリ負荷を大きく軽減します。
* **サイドボタンナビゲーション**：マウスのサイドボタンによる進む / 戻る操作に対応し、さまざまなビューでディレクトリ階層を素早く移動できます。
* **高機能パスバー**：マウスホイールによるスクロール、同階層ディレクトリのプルダウン切替、パス表示、元ルートへのジャンプに対応。全体検索と分析スイートも統一パス体系に統合されています。
* **体験強化**：スター、種類など多軸ソート、夜間モード、ワンクリックの**ぼかしカバー**に対応し、さらに **SWR の静的リフレッシュ戦略**によりビュー更新体験を向上させます。
* **バックグラウンド索引と保護**：ホームの青い点滅インジケーターでバックグラウンドのディレクトリツリー同期状態を表示。並行操作保護も内蔵し、データ汚染や競合書き込みをできるだけ防ぎます。

### 🌐 転送と共有センター

* **共有管理**：抽出回数の上限を設定可能。上限到達後は自動で共有を解除し、リンクを無効化します。
* **高速アップロード**：ローカルのファイル / フォルダを直接 Web にドラッグ＆ドロップしてアップロード可能。小ファイル大量転送時の中断率と全体の転送体験を改善します。
* **クラウドDL強化**：一括オフラインリンクの**自動重複排除**に対応。**磁力リンク自動クリーニングエンジン**を内蔵し、Base32 / Hex ハッシュを自動抽出してノイズ文字を除去します。
* **Torrent とフォールバック**：**.torrent** ファイルの解析に対応し、一部制限リンクに対しては**Web スナップショット保存**をフォールバック手段として提供します。

* **メディアモード**：設定で有効化すると、ドラマ / 漫画系ディレクトリを優先的に名前 **A-Z** 順で並べ、閲覧の連続性を高めます。

* **外部直結**：動画ストリームの直リンクをワンクリックで取得、または **PotPlayer** を起動して外部再生できます。

* **配信強化**：長時間接続の状態監視、失敗時のエラー一覧自動出力、さらに**フォルダダウンロードフィルタ**にも対応。

## 📥 インストールガイド (Installation)

1. **スクリプトマネージャーをインストール**：[Violentmonkey](https://violentmonkey.github.io/get-it/) を優先的におすすめします。[Tampermonkey](https://www.tampermonkey.net/) も利用できます。

3. **PikPak Web を開く**： [PikPak](https://mypikpak.com/drive) にアクセスしてログインします。
4. **Enhancement Master を起動**：
   * 通常モード：ログイン後、サイドバーに浮かぶ青い **PikPak Logo ボール** をクリックして起動します。
   * ターボモード：ログイン後、スクリプトが自動起動して Web 側を引き継ぎます。

## ❓ よくある質問 (FAQ)

**Q：スクリプトをインストールしたのに、青いフローティングボールが表示されません。**  
**A：以下の順に確認してください。**

1. スクリプトマネージャー内で **PikPak Enhancement Master** が有効になっていることを確認してください。
2. PikPak Web にログイン済みであることを確認してください。
3. **Tampermonkey** を使用している場合は、ブラウザ拡張機能の詳細ページで **「ユーザースクリプトを許可」** が有効になっていることを確認してください。
4. **Edge / Chrome** の最新版ブラウザの使用をおすすめします。
5. 上記をすべて確認しても表示されない場合は、他のブラウザ拡張機能を一時的にすべて無効化し、ページを再読み込み（F5）して再試行してください。

**Q：「ターボモード（Turbo Mode）」とは何ですか？ なぜ有効化が推奨されているのですか？**  
**A：** 有効化すると、スクリプトが Web 側のネイティブロジックを深く引き継ぎ、主要 UI を全画面で置き換えます。このモードでは高メモリ消費の同期処理を積極的に抑制するため、大量ファイル環境でのカクつき、クラッシュ、読み込み遅延を大きく改善できます。  
**注意：** ターボモードを有効化すると、最小化ボタンと閉じるボタンは自動で非表示になります。元の Web ページへ戻したい場合は、設定で無効にしてください。  
**おすすめ：** クラウド内のファイル数が多い場合は、より滑らかで安定した体験のため有効化を強くおすすめします。

**Q：ホームの My Pack に「デフォルト」タグが付き、スター追加・コピー・移動・リネーム・削除ができないのはなぜですか？**  
**A：** これは PikPak 公式によるデフォルトディレクトリ保護仕様です。スクリプトはその保護を維持しつつ、デフォルトフォルダ **My Pack** に対する共有制限のみ解除しています。

**Q：「ファイル重複検出」と「フォルダ重複検出」の本質的な違いは何ですか？ 実際にはどう使い分ければよいですか？**  
**A：**  
**ファイル重複検出：** 「単一ファイル」単位の重複検出で、**ハッシュの完全一致、動画長さの類似一致、名前の類似一致**の 3 モードで判定します。単話動画、単画像、個別圧縮ファイルなど、散在した重複ファイルの整理に向いています。  

**フォルダ重複検出：** 「フォルダ全体の構造」単位の重複検出で、**名前一致、内部ファイルの重なり率（類似度一致）、部分集合冗長（包含率一致）**の 3 モードで判定します。重複保存された作品一式、画像パック、階層付き資料ディレクトリなどの整理に向いています。フォルダ名が違っていても、内部ファイルが高い一致率を持つ場合や、明確な包含関係がある場合は検出できます。

> **おすすめ：**  
> ファイル重複検出の「完全一致」はハッシュベースのため最も信頼性が高く、比較的安心して一括選択・削除できます。  
> 一方、「長さ一致」「名前一致」、およびフォルダ重複検出の 3 モードはいずれも類似判定のため、一括削除の前に必ず手動確認することをおすすめします。

**Q：ファイル重複検出で見つかる重複ファイルがどんどん増えるのはなぜですか？ 全体索引の取りこぼしですか？**  
**A：** いいえ。索引の欠落でもデータ異常でもありません。重複結果のうち「完全一致」の件数は基本的に安定しています。後から重複が増えて見える主な理由は、**動画長さ**ベースの類似判定が徐々に充実していくためです。  

PikPak の一部 API は動画長さを直接返さないため、初回分析時には長さ情報が不足している重複動画が一時的にスキップされます。この問題に対応するため、スクリプトはバックグラウンドで動画メタデータを非同期に取得し、正確な長さを補完してローカルに保存します。利用時間が長くなるほどローカルの長さデータベースが充実し、**ファイル名は違うが長さが一致する**隠れた重複も見つけやすくなるため、結果が増えて見えるのです。

**Q：「画像で検索」で画像を直接送れず、貼り付けを求められることがあるのはなぜですか？**  
**A：** これは主にブラウザの **CORS 制限**によるものです。クラウド内画像を対象サービスへ直接送信できない場合、スクリプトは現在の画面を自動でクリップボードへコピーします。その場合は、開いた検索ページで `Ctrl + V` を押せば画像送信が完了します。

**Q：動画再生時に音声だけで映像が出ないことがあります。なぜですか？**  
**A：** 多くの場合、その動画が **m3u8 の多重音声 / 多トラックストリーム**だからです。クラウドトランスコード後に音声と映像のトラックが分離されることがあり、現在の Web プレーヤーではこの種の特殊ストリームを完全に解析できません。  
その場合は、右下の **「外部プレーヤーで再生」** を使って PotPlayer などのローカルプレーヤーで再生することをおすすめします。

**Q：画質メニューの「原画（高速）」と通常の「原画」の違いは何ですか？**  
**A：** 画質自体に大きな違いはほぼなく、主な違いは内部の転送経路とデコード戦略です。  
**「原画（高速）」** は、公式側のストリーミング最適化経路を利用するため、PotPlayer などの外部プレーヤー使用時に読み込みが速く、シークも滑らかで安定しやすい傾向があります。  
**おすすめ：** Web 再生でも外部再生でも、通常は **「原画（高速）」** を優先してください。

**Q：スマホアプリや公式 Web で今行った操作（アップロード、削除、移動など）が、スクリプトの分析 / 重複検出リストにすぐ反映されないのはなぜですか？**  
**A：** これは検索・分析・フィルタ速度を確保するための設計です。スクリプトは多数のファイル情報を都度再取得する代わりに、ローカルメモリスナップショットと索引キャッシュを積極的に利用しています。  
スマホアプリや公式 Web 側で多くの変更を行った直後は、ページを再読み込みして全体索引を取り直してください。  
**おすすめ：** スクリプトで分析、重複検出、整理などを実行中は、複数端末から同じファイル群を同時に変更しないでください。

**Q：一括解凍では、複数ファイルに異なるパスワードを一度に設定できますか？**  
**A：** できます。スクリプトの一括解凍は**パスワード保管庫のスマートマッチ**方式を採用しています。事前に既知のパスワードを保管庫に保存しておけば、各圧縮ファイルごとに登録済みパスワードを自動で試行し、一致した時点で解凍を開始します。

**Q：一括解凍で「システムが混雑しています / 再試行待ち」と表示されるのは、スクリプトの不具合ですか？**  
**A：** いいえ。多くの場合、PikPak のクラウド側で同時実行制限に達しているためであり、公式 API 側が混雑している状態です。スクリプト自体の不具合ではありません。  
何度試しても失敗する場合は、そのファイルがクラウド上で破損している、利用制限がかかっている、またはサーバー状態が不安定な可能性があります。時間をおいて再試行するか、ローカルにダウンロードして処理してください。

**Q：エクスポートした設定 JSON には何が含まれますか？**  
**A：** エクスポートされる JSON には、Enhancement Master 内のローカル設定データのみが含まれます。たとえば重複検出の設定、ブラック / ホワイトリスト、パスワード保管庫、一部の履歴、各種機能設定などです。  
**PikPak アカウントのパスワードは含まれません。** そのため、端末間バックアップや移行にも安心して使えます。

**Q：バックアップをインポートすると、現在の設定やリストは上書きされますか？**  
**A：** 通常は上書きされません。スクリプトはすでに**スマートマージ**方式に対応しており、インポート時はリソース管理、履歴などのリスト型データを優先的に重複排除して統合します。  
ただし、画像検索エンジンや Aria2 アドレスなど、一部の基本設定はインポート側の値が優先されるため、長期間蓄積したローカルリストや履歴は通常失われません。

**Q：一括削除時に、一部ファイルが保護されている、または削除できないのはなぜですか？**  
**A：** まず、そのファイルが**リソース管理**に記録されていないか確認してください。設定で「削除時にリソース管理の記録項目をスキップ」を有効にしている場合、一致したファイルは保護対象として削除から除外されます。  
**対処方法：**  
1. 設定でその保護ルールを無効にしてから再度削除する  
2. またはツールバー / サイドバーの **「リソース管理」** から **「今すぐクリーンアップ」** を実行し、記録項目を物理削除する

**Q：「貼り付け」を実行したのに、ファイルが一覧に表示されないのはなぜですか？**  
**A：** まず、クラウドの空き容量が十分か確認してください。PikPak 公式は容量不足時に**サイレントブロック**を行うことがあり、明確な警告なしでバックグラウンドで処理を中断する場合があります。  
そのため、貼り付け後にファイルが現れない最も一般的な原因は容量制限です。空き容量を確保してから再試行してください。

**Q：クラウドDLダイアログの「自動修復防屏蔽磁链」とは何ですか？ いつ有効にするべきですか？**  
**A：** この機能は、磁力リンクに混ざった中国語、絵文字、記号などのノイズを自動除去し、特徴コードを抽出して標準形式へ戻します。  
**おすすめ：** 基本的には常時オンのままで問題ありません。SNS、掲示板、チャットなどで文字混入やマスク処理、意図的な汚染が入った磁力リンクに特に有効です。  
すでに標準形式で `magnet:?` から始まる正常なリンクは自動でスキップされるため、通常ダウンロードには影響しません。

**Q：「フォルダダウンロードフィルタ」とは何ですか？ どのダウンロード方法に影響しますか？**  
**A：** フォルダ全体をダウンロードする際に、**拡張子や名前ルールで一致するファイルを自動除外**する機能です。たとえば `.txt`、`.jpg`、または特定キーワードを含むファイル名を除外できます。通常のフォルダダウンロードだけでなく、**Aria2 / Motrix へのフォルダ送信**にも適用されます。

**Q：Aria2 / Motrix やブラウザでクラウドのフォルダをダウンロードする場合、階層構造は保持されますか？**  
**A：**  
**Aria2 / Motrix：保持されます。**  
スクリプトは送信時にクラウド内のディレクトリツリーを自動で複製・復元するため、ダウンロード後もローカル側のフォルダ構造はできるだけクラウドと同じになります。また、Windows 非対応文字（`:` `*` `?` など）の自動クリーニングや、長すぎるパスの短縮も行います。  

**ブラウザ標準ダウンロード：保持されません。**  
ブラウザ側のダウンロード仕様により、フォルダ内のファイルは同じディレクトリへ“平坦化”されることが一般的です。  

**おすすめ：** 複雑なディレクトリ階層を保ったままダウンロードしたい場合は、**Aria2 / Motrix への送信**を優先してください。

**Q：「複数アカウントデータ移行」とは何ですか？ どう操作すればよいですか？**  
**A：** 現在のアカウント内のファイルやフォルダを、別の PikPak アカウントへ素早くシームレスに転送できる機能です。  
**操作手順：**  
1. 現在のアカウントで移行したいファイルまたはフォルダを選択  
2. 下部の **「データ移行」** ボタンをクリック  
3. スクリプトがデータを暗号化パッケージ化し、現在のアカウントを自動ログアウト  
4. その後、転送先アカウントへ通常通りログイン  
5. ログイン後、スクリプトがローカルキャッシュの移行パッケージを自動検出し、ワンクリック受信を案内

## 🛡️ プライバシーとセキュリティ (Privacy & Security)

* **ローカル優先**：本スクリプトの中核機能はすべて、ブラウザから PikPak 公式 API と直接やり取りして動作します。アカウント Token、パスワード保管庫、大半のローカル設定データは、既定でブラウザのローカル環境に保存されます。
* **ゼロ収集**：本スクリプトは**ユーザーの個人情報を能動的に収集せず**、**ファイル情報や認証情報を第三者サーバーへ送信することもありません**。
* **第三者インターフェース**：**オンライン字幕検索**や**画像で検索**などの拡張機能を利用した場合に限り、必要最小限の検索キーワードまたは画像特徴パラメータのみを、関連する公開サービスへ送信します。個人を特定できる情報は含まれません。
