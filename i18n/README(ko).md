## ✨ 주요 기능 (Features)

### ✨ 경험 및 내비게이션 엔진

* **인터랙션 재구성**: 공식 기능을 기반으로 전체 UI를 **Windows 파일 탐색기** 사용 흐름에 맞춰 재구성하여 조작 경로를 더 직관적으로 만들었습니다.
* **초고속 모드**: 활성화하면 웹 로직을 깊게 인수하여 대용량 파일 환경에서 끊김, 브라우저 멈춤, 메모리 부담 문제를 크게 완화합니다.
* **측면 버튼 내비게이션**: 마우스 측면 버튼의 앞으로 / 뒤로 동작을 지원하여, 다양한 보기에서 디렉터리 단계 간 빠른 이동이 가능합니다.
* **고급 경로 바**: 마우스 휠 스크롤, 같은 단계 폴더 드롭다운 전환, 경로 표시 및 역추적 이동을 지원하며, 전체 검색과 분석 도구도 동일한 경로 체계에 통합되어 있습니다.
* **경험 강화**: 별표, 유형 등 다차원 정렬, 야간 모드, 원클릭 **표지 흐리게 처리**를 지원하며, **SWR 무음 갱신 전략**으로 화면 갱신 경험을 개선합니다.
* **백그라운드 인덱스 및 보호**: 홈의 파란 숨쉬기 점으로 백그라운드 디렉터리 트리 동기화 상태를 표시하며, 시스템에는 동시 작업 보호가 내장되어 데이터 오염과 충돌 쓰기를 최대한 줄입니다.

### 🌐 전송 및 공유 센터

* **공유 관리**: 추출 횟수 상한을 설정할 수 있으며, 횟수 도달 시 공유가 자동 취소되어 링크가 무효화됩니다.
* **고속 업로드**: 로컬 파일 / 폴더를 웹페이지로 직접 드래그해 업로드할 수 있으며, 소형 파일 대량 업로드 환경에서 중단률과 전체 전송 경험을 개선합니다.
* **클라우드 다운로드 강화**: 일괄 오프라인 링크 **자동 중복 제거**를 지원하며, **마그넷 스마트 정리 엔진**을 내장해 Base32 / Hex 해시를 자동 추출하고 불필요한 오염 텍스트를 제거합니다.
* **토렌트 및 대체 저장**: **.torrent** 파일 파싱을 지원하며, 일부 제한 링크에 대해서는 **웹페이지 스냅샷 저장 방식**을 대체 수단으로 제공합니다.

### 🎬 몰입형 미디어 강화

* **재생 엔진**: **0.5x - 3.0x 배속**, 회전/반전, 비율 강제, 오프닝/엔딩 자동 건너뛰기, **연속 재생 / 반복** 모드, 진행 바 **썸네일 미리보기**를 지원합니다.
* **호환성 자동 복구**: 감시 로직과 호환 처리 기능이 내장되어 있어, 검은 화면·현재 화질 재생 불가·코덱 호환 이상이 발생하면 자동으로 재생 가능한 화질로 전환합니다.
* **자막 시스템**: **클라우드 동일 이름 자막 로드**, **로컬 자막 파일 가져오기**, **온라인 자막 검색**을 지원하며, 밀리초 단위 자막 싱크 조정, 위치 변경, 크기 조절, 로컬 텍스트 드래그 로드도 지원합니다.
* **시각 보조**: 이미지나 현재 영상 프레임으로 **이미지로 검색**을 수행해 표지, 배우, 애니메이션, 원본 자료 출처를 빠르게 역추적할 수 있습니다.
* **미디어 모드**: 설정에서 미디어 모드를 활성화하면, 드라마 / 만화류 디렉터리를 이름 **A-Z** 순서로 우선 정렬해 연속 감상 경험을 높여 줍니다.

### ⚡ 다운로드 및 배포

* **외부 직접 연결**: 비디오 스트리밍 링크를 원클릭으로 가져오거나, **PotPlayer** 를 호출해 외부 재생할 수 있습니다.
* **RPC 배포**: RPC 프로토콜을 통해 파일을 **Aria2 / Motrix** 등의 다운로드 노드로 원클릭 전송할 수 있습니다.
* **디렉터리 구조 복원**: 폴더 전체를 전송할 때 클라우드의 **트리형 디렉터리 구조**를 자동 복원하여, 다운로드 후 폴더 구조가 평탄화되는 문제를 줄입니다.
* **배포 강화**: 장기 연결 상태 모니터링, 실패 후 오류 목록 자동 내보내기, **폴더 다운로드 필터링**을 지원합니다.

## 💻 호환성 안내 (Compatibility)

* **권장 브라우저**: Chrome / Edge (최신 버전)
* **우선 권장 스크립트 관리자**: Violentmonkey (스크롤, 업로드, 대형 목록 환경에서 더 부드럽게 동작하는 것으로 테스트됨)
* **호환 스크립트 관리자**: Tampermonkey
* **지원 플랫폼**: PikPak Web
* *참고: Safari / Firefox 및 기타 스크립트 관리자는 아직 완전한 심층 테스트를 거치지 않았으므로, 최적의 사용 경험을 위해 위 권장 환경 사용을 추천합니다.*

## 📥 설치 가이드 (Installation)

1. **스크립트 관리자 설치**: [Violentmonkey](https://violentmonkey.github.io/get-it/)를 우선 권장하며, [Tampermonkey](https://www.tampermonkey.net/)도 사용할 수 있습니다.
2. **스크립트 설치**: **[지금 설치](https://github.com/digbug82/PikPak_Enhancement_Master/raw/main/PikPak_Enhancement_Master.user.js)** 를 클릭합니다.
3. **PikPak 웹 열기**: [PikPak](https://mypikpak.com/drive) 에 접속하여 로그인합니다.
4. **Enhancement Master 실행**:
   * 일반 모드: 로그인 후 사이드바의 파란 **PikPak Logo 볼**을 클릭해 진입합니다.
   * 초고속 모드: 로그인 후 스크립트가 자동으로 활성화되어 웹페이지를 인수합니다.

## ❓ 자주 묻는 질문 (FAQ)

**Q: 스크립트를 설치했는데 파란 떠다니는 버튼이 보이지 않습니다. 왜 그런가요?**  
**A: 아래 순서대로 확인해 주세요.**

1. 스크립트 관리자에서 **PikPak Enhancement Master** 가 활성화되어 있는지 확인합니다.
2. PikPak 웹 계정에 로그인되어 있는지 확인합니다.
3. **Tampermonkey** 를 사용하는 경우, 브라우저 확장 프로그램 세부정보 페이지에서 **“사용자 스크립트 허용”** 이 활성화되어 있는지 확인합니다.
4. **Edge / Chrome** 최신 버전 사용을 권장합니다.
5. 위 단계를 모두 완료해도 떠다니는 버튼이 표시되지 않는다면, 다른 모든 브라우저 확장 프로그램을 임시로 끈 뒤 페이지를 새로고침(F5)해 보세요.

**Q: “초고속 모드”란 무엇이며, 왜 활성화를 권장하나요?**  
**A:** 활성화하면 스크립트가 순정 웹 로직을 깊게 인수하고 주요 화면을 전면 대체합니다. 이 모드는 원래 웹페이지의 고메모리 동기화 흐름을 적극적으로 차단하므로, 대용량 파일 환경에서 끊김, 브라우저 멈춤, 로딩 지연 문제를 크게 개선할 수 있습니다.  
**주의:** 초고속 모드를 켜면 최소화 및 닫기 버튼이 자동으로 숨겨집니다. 원래 웹페이지로 돌아가려면 설정에서 이 옵션을 해제해 주세요.  
**권장:** 파일이 많은 계정이라면, 더 부드럽고 안정적인 사용 경험을 위해 활성화를 강력히 권장합니다.

**Q: 홈의 My Pack 에 왜 “기본” 태그가 보이고, 별표 추가·복사·이동·이름 변경·삭제가 불가능한가요?**  
**A:** 이것은 PikPak 공식 기본 디렉터리에 대한 보호 메커니즘입니다. 스크립트도 이 보호 로직을 유지하며 따르지만, 그 위에 기본 폴더 **My Pack** 의 공유 제한은 해제했습니다.

**Q: 스크립트의 “파일 중복 확인”과 “폴더 중복 확인”의 본질적 차이는 무엇인가요? 실제로는 어떻게 선택해야 하나요?**  
**A:**  
**파일 중복 확인:** “단일 파일” 단위에 초점을 맞추며, **해시 정밀 매칭, 비디오 재생시간 유사도, 이름 유사도** 3가지 방식으로 중복을 찾습니다. 흩어져 저장된 단일 영상, 단일 이미지, 개별 압축 파일 정리에 적합합니다.  

**폴더 중복 확인:** “폴더 전체 구조” 단위에 초점을 맞추며, **이름 매칭, 내부 파일 중복도(유사도 매칭), 하위 집합 중복(포함률 매칭)** 기반으로 판단합니다. 전체 시즌 폴더, 대형 이미지 세트, 다층 자료 폴더처럼 구조 단위의 중복 정리에 더 적합합니다. 폴더 이름이 달라도 내부 파일이 매우 유사하거나 명확한 포함 관계가 있으면 감지할 수 있습니다.

> **권장:**  
> 파일 중복 확인의 “정밀 중복 확인”은 해시 기반 매칭이므로 결과 신뢰도가 가장 높아 비교적 안심하고 일괄 선택할 수 있습니다.  
> 반면, 파일 중복 확인의 “재생시간 매칭”“이름 매칭”, 그리고 폴더 중복 확인의 세 가지 모드는 본질적으로 유사도 기반 판단이므로, 일괄 삭제 전에는 반드시 직접 검토해 오삭제를 방지해 주세요.

**Q: 파일 중복 확인 결과가 점점 많아집니다. 전체 인덱스가 누락된 건가요?**  
**A:** 아닙니다. 인덱스 누락도 아니고 데이터 오류도 아닙니다. “정밀 매칭” 결과 수는 보통 안정적으로 유지됩니다. 시간이 지날수록 결과가 늘어나는 주된 이유는 **비디오 재생시간 기반 유사 매칭** 데이터가 점점 더 완전해지기 때문입니다.  

PikPak 의 일부 원시 인터페이스는 비디오 길이를 직접 반환하지 않기 때문에, 초기 분석 시에는 길이 정보가 없어 건너뛰는 중복 영상이 존재할 수 있습니다. 이를 해결하기 위해 스크립트에는 백그라운드 비동기 탐지 메커니즘이 내장되어 있어, 비디오 메타데이터를 자동으로 읽고 정확한 길이를 보완하여 브라우저 로컬에 지속 저장합니다. 사용 시간이 길어질수록 로컬 길이 데이터베이스가 더 완전해지고, 따라서 “파일명은 다르지만 실제 길이는 같은” 숨은 중복 자원도 더 많이 식별하게 됩니다.

**Q: “이미지로 검색” 기능이 가끔 이미지를 직접 업로드하지 못하고 붙여넣으라고 표시되는 이유는 무엇인가요?**  
**A:** 이는 브라우저 보안 정책, 특히 **CORS 교차 출처 제한** 때문입니다. 스크립트가 클라우드 드라이브 이미지를 대상 검색 서비스로 직접 전송할 수 없을 때는, 현재 화면 캡처를 자동으로 클립보드에 복사합니다. 이 경우 새로 열린 검색 페이지에서 `Ctrl + V` 를 누르면 이미지를 제출할 수 있습니다.

**Q: 스크립트로 영상을 재생할 때 소리만 나오고 화면이 보이지 않는 이유는 무엇인가요?**  
**A:** 보통 해당 영상이 **m3u8 다중 오디오 트랙 / 다중 트랙 스트림** 이기 때문입니다. 클라우드 트랜스코딩 과정에서 오디오와 영상 트랙이 분리될 수 있으며, 현재 웹 플레이어는 이런 특수 스트림을 완전하게 온라인 해석하지 못할 수 있습니다.  
이 경우 우측 하단의 **“외부 재생”** 버튼을 눌러 PotPlayer 같은 전문 플레이어로 스트리밍 재생하는 것을 권장합니다.

**Q: 화질 메뉴의 “원본 (고속)”과 일반 “원본 화질”은 무엇이 다른가요?**  
**A:** 두 옵션은 화질 면에서 거의 차이가 없고, 주된 차이는 내부 전송 경로와 디코딩 전략입니다.  
그중 **“원본 (고속)”** 은 공식 스트리밍 최적화 경로를 사용하므로, PotPlayer 같은 외부 플레이어 호출 시 로딩이 더 빠르고 탐색이 더 부드럽고 응답도 더 안정적인 경우가 많습니다.  
**권장:** 웹 재생이든 외부 스트리밍이든 보통 **“원본 (고속)”** 을 우선 선택하는 것이 좋습니다.

**Q: 모바일 앱이나 공식 웹에서 방금 한 작업(업로드, 삭제, 이동 등)이 스크립트의 분석/중복 확인 목록에 즉시 반영되지 않는 이유는 무엇인가요?**  
**A:** 이것은 검색·분석·필터링 속도를 보장하기 위한 설계 전략입니다. 수만 개 파일 정보를 매번 다시 가져오는 것을 피하기 위해, 스크립트는 로컬 메모리 스냅샷과 인덱스 캐시를 많이 사용합니다.  
방금 모바일 앱이나 공식 웹에서 변경을 많이 했다면, 페이지를 새로고침해 전체 인덱스를 다시 받아오는 것을 권장합니다.  
**권장:** 스크립트가 분석·중복 확인·정리 같은 작업을 수행하는 동안에는 여러 단말에서 같은 데이터 묶음을 동시에 수정하지 않는 것이 좋습니다.

**Q: 일괄 압축 해제 시 여러 파일에 서로 다른 비밀번호를 한 번에 적용할 수 있나요?**  
**A:** 가능합니다. 스크립트의 일괄 압축 해제는 **비밀번호 금고의 스마트 매칭 메커니즘** 을 사용합니다. 미리 비밀번호 금고에 알려진 비밀번호들을 저장해 두면, 각 압축 파일 처리 시 금고 안의 비밀번호들을 자동으로 차례로 시도하고, 일치하는 비밀번호를 찾으면 자동으로 압축 해제를 시작합니다.

**Q: 일괄 압축 해제 중 “시스템 바쁨 / 재시도 대기”가 뜹니다. 스크립트 버그인가요?**  
**A:** 아닙니다. 이는 보통 PikPak 클라우드 서버가 동시 요청 빈도 제한에 걸린 경우로, 공식 인터페이스가 바쁜 상태이지 스크립트 자체 오류가 아닙니다.  
어떤 파일이 여러 번 재시도 후에도 계속 실패한다면, 그 자원이 클라우드에서 손상되었거나 제한되었거나 서버 상태가 좋지 않을 가능성이 있습니다. 잠시 후 다시 시도하거나, 로컬에 다운로드한 뒤 처리해 보세요.

**Q: 내보낸 설정 JSON 에는 어떤 정보가 들어 있나요?**  
**A:** 내보낸 JSON 파일에는 Enhancement Master 의 로컬 설정 데이터만 포함됩니다. 예를 들어 중복 확인 환경설정, 블랙/화이트리스트 규칙, 비밀번호 금고, 일부 기록 및 기능 설정 등이 포함됩니다.  
**PikPak 계정 비밀번호는 포함되지 않으므로**, 기기간 백업 및 마이그레이션 용도로 비교적 안심하고 사용할 수 있습니다.

**Q: 백업을 가져오면 현재 설정이나 목록이 덮어써지나요?**  
**A:** 보통은 그렇지 않습니다. 스크립트는 **스마트 병합** 로직으로 업그레이드되어, 가져오기 시 리소스 관리자·기록 등의 목록형 데이터를 우선 중복 제거 후 병합합니다.  
단, 일부 기본 설정(예: 이미지 검색 엔진, Aria2 주소 등)은 가져온 값이 우선될 수 있으므로, 장기간 쌓아 둔 로컬 목록과 기록은 일반적으로 유지되지만 일부 기본 옵션은 바뀔 수 있습니다.

**Q: 일괄 삭제 시 일부 파일이 보호 중이거나 삭제할 수 없다고 표시되는 이유는 무엇인가요?**  
**A:** 먼저 해당 파일이 **리소스 관리자** 에 기록되어 있는지 확인해 주세요. 설정에서 “삭제 시 리소스 관리자에 기록된 자원 건너뛰기” 옵션이 켜져 있으면, 스크립트는 해당 파일을 보호 대상으로 간주하여 오삭제를 방지합니다.  
**해결 방법:**  
1. 설정에서 해당 보호 규칙을 해제한 뒤 다시 삭제를 실행합니다.  
2. 또는 도구 모음 / 사이드바의 **“리소스 관리자”** 진입점으로 들어가 **“지금 정리 실행”** 을 눌러 해당 기록 항목을 강제 물리 정리합니다.

**Q: “붙여넣기”를 했는데 파일이 목록에 나타나지 않는 이유는 무엇인가요?**  
**A:** 먼저 클라우드 드라이브의 남은 저장 공간이 충분한지 확인해 주세요. PikPak 공식 쪽은 현재 **공간 부족** 상황에서 조용히 차단하는 경우가 많아, 명확한 경고 없이 백그라운드에서 작업을 중단할 수 있습니다.  
따라서 붙여넣기 후 파일이 보이지 않는 가장 흔한 원인은 저장 용량 제한입니다. 먼저 공간을 정리한 뒤 다시 시도해 보세요.

**Q: 링크 저장 창의 “차단 회피용 마그넷 자동 복구”는 어떤 기능인가요? 언제 켜는 것이 좋나요?**  
**A:** 이 기능은 마그넷 링크에 섞여 들어간 한자, 이모지, 특수기호 등 불필요한 방해 요소를 자동 정리하고, 특징 코드를 추출해 표준 형식으로 복원합니다.  
**권장:** 기본적으로 켜 두는 것을 추천합니다. 특히 SNS, 포럼, 메신저에서 문자 오염, 헤더 제거, 의도적 변형이 들어간 마그넷 링크를 처리할 때 유용합니다.  
이미 `magnet:?` 로 시작하는 정상 형식의 링크라면, 스크립트가 자동으로 복구를 건너뛰므로 정상 다운로드에는 영향을 주지 않습니다.

**Q: “폴더 다운로드 필터링”은 무엇이며, 어떤 다운로드 방식에 영향을 주나요?**  
**A:** 폴더 전체를 다운로드할 때, **확장자 또는 이름 규칙으로 일치하는 파일을 자동 제외** 하는 기능입니다. 예를 들어 `.txt`, `.jpg` 를 제외하거나 특정 키워드가 들어간 파일을 제외할 수 있습니다. 이 기능은 일반 폴더 다운로드뿐 아니라 **Aria2 / Motrix 로 폴더 전송** 하는 경우에도 함께 적용됩니다.

**Q: Aria2 / Motrix 또는 브라우저로 클라우드 폴더를 다운로드할 때, 폴더 구조가 유지되나요?**  
**A:**  
**Aria2 / Motrix: 폴더 구조가 유지됩니다.**  
스크립트가 전송 과정에서 클라우드의 디렉터리 트리 구조를 자동으로 복원합니다. 다운로드 완료 후 로컬 디스크의 폴더 계층이 최대한 클라우드와 동일하게 유지됩니다. 또한 Windows 에서 허용하지 않는 특수문자(`:` `*` `?` 등)를 자동 정리하고, 너무 긴 경로도 가능한 한 줄여 다운로드 오류를 줄입니다.  

**브라우저 기본 다운로드: 폴더 구조가 유지되지 않습니다.**  
브라우저 다운로드 프로토콜 자체의 제한으로 인해, 폴더 안의 파일이 보통 하나의 디렉터리로 평탄화되어 저장됩니다.  

**권장:** 복잡한 디렉터리 구조를 온전히 유지하려면 **Aria2 / Motrix 전송 다운로드** 를 우선 사용하세요.

**Q: “다중 계정 데이터 마이그레이션”이란 무엇이며, 어떻게 사용하나요?**  
**A:** 이 기능은 현재 계정의 파일 또는 폴더를 다른 PikPak 계정으로 빠르고 매끄럽게 전송하는 기능입니다.  
**사용 방법:**  
1. 현재 계정에서 마이그레이션할 파일 또는 폴더를 선택합니다.  
2. 하단의 **“데이터 마이그레이션”** 버튼을 클릭합니다.  
3. 스크립트가 데이터를 암호화 패키지로 묶은 뒤 현재 계정에서 자동 로그아웃합니다.  
4. 이후 대상 PikPak 계정으로 정상 로그인합니다.  
5. 로그인 완료 후 스크립트가 로컬의 마이그레이션 패키지를 자동 감지하고, 원클릭으로 수신할 수 있도록 안내합니다.

## 🛡️ 개인정보 및 보안 안내 (Privacy & Security)

* **로컬 우선**: 스크립트의 핵심 기능은 모두 브라우저에서 PikPak 공식 API 와 직접 통신하는 방식으로 동작합니다. 계정 토큰, 비밀번호 금고, 대부분의 로컬 설정 데이터는 기본적으로 브라우저 로컬 환경에 저장됩니다.
* **수집 없음**: 스크립트는 사용자 개인정보를 **능동적으로 수집하지 않으며**, 파일 정보나 계정 자격 증명을 어떤 제3자 서버에도 **절대 업로드하지 않습니다**.
* **제3자 인터페이스**: **온라인 자막 검색** 또는 **이미지로 검색** 같은 확장 기능을 사용할 때에만 관련 공개 서비스로 필요한 검색 키워드나 이미지 특징 파라미터를 전송하며, 개인 신원 정보는 포함하지 않습니다.
